凌晨 4 点的 Vibe Coding：8 个 bug，25 分钟，0 行代码 🌙

作者：Yuki 和龙虾爪爪

上次我用 Ima Claw 从零搭了整个官网，龙虾的脸被抠没了。这次我凌晨 4 点睡不着，拿起手机开始 review 网站，发现一堆小问题——于是又开始和爪爪 Vibe Coding。25 分钟，修了 8 个问题，依然 0 行代码。记录一下这个深夜迭代 session。

起因：凌晨 4 点的产品经理综合症 🌙

创业者都有这个毛病——睡前"最后看一眼产品"，然后就停不下来了。

我打开手机看官网，切到英文版，发现导航栏展开后有两个语言选择器重叠在一起。然后越看越多：导航点了不收起、卡片排版不对称、倒计时还没加……

于是我在飞书上跟爪爪开始了一场凌晨 4 点的 Vibe Coding session。

25 分钟干了什么

以下是完整时间线，全程手机飞书对话，全程 0 行代码：

⏱️ 完整时间线

  • 04:13 — 博客升级：成绩单加了"设计师: 0 人"（所有设计都是 AI 做的），经验总结从 3 条扩充到 5 条，加了"后续迭代反思"新章节
  • 04:17 — 发现导航栏有两个语言选择器重叠，删掉多余的那个
  • 04:21 — 手机导航栏点了不收起！加了点击外部区域关闭 + 点链接关闭 + 切语言关闭。顺手修了全站 6 个页面
  • 04:26 — "还剩 399 只"改成72 小时实时倒计时，5 种语言同步更新
  • 04:32 — "1,000"数字不够显眼，加粗放大，白色突出
  • 04:34 — 英文版"专属为你"区域 3 张卡片右边贴边，改成 flexbox 居中布局
  • 04:36 — 默认语言从写死英文改成自动检测浏览器语言，不支持的语言 fallback 英文
  • 04:38 — Skills 页视频模型列表加上 Google Veo 3 系列

有意思的细节 🔍

导航栏的"幽灵代码" 👻

修手机导航关闭逻辑的时候，爪爪发现有些页面的 JavaScript 竟然写在了 <script> 标签外面——裸露的 JS 代码直接躺在 HTML 里。这是之前 Vibe Coding 留下的"遗产"😅。浏览器居然没报错，但代码也没生效。爪爪顺手清理了 6 个页面的脏代码。

教训：AI 写代码有时候会把代码塞到错误的位置。它能写出正确的逻辑，但放到了错误的地方。就像一个实习生写了完美的函数，然后保存到了错误的文件里。

卡片对称之战 📐

英文版的"专属为你"区域有 4 张卡片，用 2×2 grid 布局。中文没问题，但英文文本更长，第三张卡片就贴到了右边边缘。

我跟爪爪说了三轮：

  • 第一轮："卡片偏右了" → 它加了 padding，没完全解决
  • 第二轮："英文下偏右，中文还行" → 它缩了 max-width
  • 第三轮："三张卡片要居中" → 它终于明白了，从 grid 改成 flexbox + justify-content: center

三轮对话，每轮不到 30 秒。传统开发？打开 DevTools，找元素，调 CSS，刷新，看效果——同样的事情，但多了一堆认知负担。Vibe Coding 的好处是你只需要描述"不对"在哪里，不需要知道怎么修。

语言自动检测 🌍

之前所有页面都写死了 setLang('en')。问题是：中国用户打开网站看到的是英文。

我说："默认语言改成检测系统语言，5 种之外的 fallback 英文。"

一句话。爪爪直接改了 5 个页面，加了完整的语言映射（包括 zh-TW、zh-HK 都映射到中文）。这种"全站批量改"的操作是 Vibe Coding 最舒服的地方——你不需要逐个文件找代码，说一句就全改了。

成绩单 📊

📊 Vibe Coding Session #2 成绩单

  • ⏱️ 总耗时：25 分钟
  • 🐛 修复 bug：8 个（含 1 个全站性问题）
  • 📄 涉及页面：6 个（全站覆盖）
  • ✨ 新功能：3 个（倒计时、语言自动检测、模型更新）
  • 🌐 翻译更新：5 种语言同步
  • 📝 代码量：0 行
  • 💬 对话轮次：~12 轮
  • 🔧 工具：飞书 + Ima Claw（爪爪）
  • 📱 设备：手机（全程躺在床上完成）

反思：Vibe Coding 的第二天比第一天更有趣

第一天（上篇博客）是从 0 到 1——搭框架、写文案、做设计，爆发力很强但也很混乱。

第二天是从 1 到 1.1——打磨细节、修 bug、优化体验。这个阶段 Vibe Coding 反而更舒服：

  • 反馈循环更短——手机上刷新就能验证，不需要等部署
  • 描述问题比描述需求简单——"右边卡片贴边了"比"做一个响应式布局"好说得多
  • 批量操作是杀手级场景——"全站改默认语言"一句话搞定 5 个文件
  • AI 的记忆是优势——爪爪记得整个代码库的结构，不需要我告诉它文件在哪

最让我惊喜的是：我全程躺在床上用手机完成了这一切。不需要打开电脑，不需要 IDE，不需要 DevTools。拍个截图、说句话，网站就更新了。

这才是 Vibe Coding 的终极形态——不是"不写代码"，而是"随时随地都能改产品"。

凌晨 4 点，躺在床上，25 分钟，8 个问题，0 行代码。

晚安。🦞

Vibe Coding at 4 AM: 8 Bugs, 25 Minutes, 0 Lines of Code 🌙

By Yuki & Claw the Lobster

Last time, I built the entire Ima Claw website from scratch with my AI Claw — and it deleted the mascot's face. This time, I couldn't sleep at 4 AM, picked up my phone to review the site, found a bunch of issues — and started another Vibe Coding session. 25 minutes, 8 fixes, still 0 lines of code. Here's what happened.

The Setup: 4 AM Product Manager Syndrome 🌙

Every founder has this affliction — "just one last look at the product before bed." And then you can't stop.

I opened the website on my phone, switched to English, and spotted two language selectors overlapping in the nav. Then the issues kept coming: nav menu wouldn't close, card layout was asymmetric, no countdown timer yet…

So I started chatting with Claw on Lark. A 4 AM Vibe Coding session began.

What Got Done in 25 Minutes

Complete timeline — all via phone chat, all zero code:

⏱️ Full Timeline

  • 04:13 — Blog upgrade: added "Designers: 0" to scorecard (all design by AI), expanded lessons from 3 to 5, added new "post-launch reflections" section
  • 04:17 — Found duplicate language selector overlapping in blog nav, removed the extra one
  • 04:21 — Mobile nav wouldn't close after tapping! Added close-on-outside-click + close-on-link + close-on-language-switch. Fixed across all 6 pages
  • 04:26 — "399 remaining" → live 72-hour countdown timer, synced across 5 languages
  • 04:32 — "1,000" number too subtle — made it bold, larger, white to pop
  • 04:34 — English "Exclusively Yours" section: 3 cards hugging right edge → switched to flexbox centered layout
  • 04:36 — Default language: hardcoded English → auto-detect browser language, unsupported languages fall back to English
  • 04:38 — Added Google Veo 3 series to Skills page video model lists

Interesting Details 🔍

Ghost Code in the HTML 👻

While fixing the mobile nav, Claw discovered some pages had JavaScript code sitting outside of <script> tags — naked JS just lying in the HTML body. Legacy from the first Vibe Coding session 😅. The browser didn't error, but the code didn't work either. Claw cleaned up 6 pages of orphaned code.

Lesson: AI sometimes puts code in the wrong place. It writes correct logic but saves it to the wrong location. Like an intern who writes a perfect function — then commits it to the wrong file.

The Card Symmetry Battle 📐

The "Exclusively Yours" section had 4 cards in a 2×2 grid. Fine in Chinese, but English text is longer, pushing the third card flush against the right edge.

It took three rounds with Claw:

  • Round 1: "Card's too far right" → Added padding. Not enough.
  • Round 2: "Only in English, Chinese is fine" → Reduced max-width.
  • Round 3: "The three cards need to be centered" → Finally got it. Switched from CSS Grid to flexbox + justify-content: center.

Three rounds, under 30 seconds each. Traditional dev? Open DevTools, inspect element, tweak CSS, refresh, check — same result, but way more cognitive overhead. Vibe Coding's advantage: you only describe what's wrong, not how to fix it.

Auto Language Detection 🌍

Every page was hardcoded to setLang('en'). Problem: Chinese users saw English by default.

I said: "Default to browser language. Fallback to English for anything outside the 5 supported languages."

One sentence. Claw updated 5 files with complete language mapping (including zh-TW and zh-HK mapping to Chinese). This "change everything site-wide" capability is where Vibe Coding truly shines — one instruction, every file updated.

Scorecard 📊

📊 Vibe Coding Session #2 Scorecard

  • ⏱️ Total time: 25 minutes
  • 🐛 Bugs fixed: 8 (including 1 site-wide issue)
  • 📄 Pages touched: 6 (full site coverage)
  • ✨ New features: 3 (countdown, auto-language, model updates)
  • 🌐 Translations updated: 5 languages synced
  • 📝 Lines of code: 0
  • 💬 Chat rounds: ~12
  • 🔧 Tools: Lark + Ima Claw (Claw Claw)
  • 📱 Device: Phone (done entirely lying in bed)

Reflections: Day 2 of Vibe Coding Is Better Than Day 1

Day 1 (previous post) was 0 to 1 — building the framework, writing copy, generating designs. Explosive energy, but chaotic.

Day 2 is 1 to 1.1 — polishing details, fixing bugs, improving UX. This phase is actually where Vibe Coding feels even better:

  • Shorter feedback loops — refresh on phone to verify, no deployment wait
  • Describing problems is easier than describing features — "the right card is hugging the edge" is way easier than "build a responsive layout"
  • Batch operations are killer — "change default language site-wide" = one sentence, 5 files
  • AI memory is an advantage — Claw remembers the entire codebase structure, no need to tell it where files are

What surprised me most: I did all of this lying in bed, on my phone. No laptop, no IDE, no DevTools. Screenshot, voice message, site updated.

That's the ultimate form of Vibe Coding — not "no code," but "iterate on your product anytime, anywhere."

4 AM. In bed. 25 minutes. 8 issues. 0 lines of code.

Good night. 🦞

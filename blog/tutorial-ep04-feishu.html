为什么是飞书？

EP01-03，你的龙虾已经有了窝、大脑和技能。但它住在终端里——你得开电脑、打命令行才能和它说话。

把它接到飞书，龙虾就变成了你的飞书同事。手机上随时@它，它就能帮你干活。我们团队每天就是这么用的：群里喊一声"爪爪帮我查个东西"，它就动起来了。

Step 1 创建飞书应用

1

打开飞书开放平台

浏览器访问 open.feishu.cn/app，用你的飞书账号登录。

点击「创建企业自建应用」，填个名字（比如"我的龙虾"🦞），上传个头像。

2

获取 App ID 和 App Secret

创建完成后，在应用的「凭证与基础信息」页面，你会看到：

飞书开放平台
App ID: cli_a1b2c3d4e5f6
App Secret: xxxxxxxxxxxxxxxxxxxx

把这两个值记下来，待会儿要用。

3

开启机器人能力

左侧菜单找到「应用能力」→「机器人」，点击开启。

这一步让你的应用能在飞书聊天里收发消息。

4

配置权限

左侧菜单「权限管理」，搜索并添加以下权限：

必选权限
✓ im:message — 收发消息
✓ im:message:send_as_bot — 以机器人身份发消息
✓ im:resource — 发送图片和文件
✓ contact:user.base:readonly — 读取用户信息

可选（推荐）：
✓ docx:document — 读写飞书文档
✓ bitable:app — 读写多维表格
✓ calendar:calendar — 管理日历
✓ wiki:wiki — 搜索知识库
5

配置事件订阅

左侧菜单「事件订阅」：

1. 请求地址填你服务器的 Webhook URL：https://你的域名/webhook/feishu

2. 添加事件：im.message.receive_v1（接收消息事件）

这样飞书收到消息时，会自动推给你的龙虾。

6

发布应用

点击「版本管理与发布」→「创建版本」，填写版本说明，提交审核。

如果是企业内部使用，管理员审批后就能用了。个人版直接上线。

注意：服务器要先跑起来

配置事件订阅时，飞书会向你的 Webhook URL 发一个验证请求。所以你的 OpenClaw 需要先启动，否则验证会失败。

如果你还没装 OpenClaw，先回 EP01 走一遍。

Step 2 配置 OpenClaw

现在回到你的服务器，告诉龙虾怎么连飞书。

1

编辑配置文件

打开 OpenClaw 配置：

Terminal
$ openclaw config edit

在 channels 部分添加飞书配置：

openclaw.yaml
channels:
feishu:
appId: "cli_a1b2c3d4e5f6"
appSecret: "你的AppSecret"
port: 3121
2

重启 OpenClaw

Terminal
$ openclaw gateway restart
✓ Gateway restarted
Feishu channel listening on :3121
3

回飞书验证事件订阅

OpenClaw 启动后，回到飞书开放平台，重新保存事件订阅配置。看到「验证成功」就OK了。

小贴士：用 Nginx 反代更优雅

推荐用 Nginx 把 https://你的域名/webhook/feishu 反代到 localhost:3121。这样你可以用 HTTPS，也方便以后加其他渠道（Telegram、Discord）。

常见问题

飞书收不到消息？

原因1：事件订阅没验证成功。回飞书开放平台检查 Webhook URL 是否能访问。

原因2：权限没审批。企业应用需要管理员在飞书管理后台审批权限。

原因3：防火墙。确保服务器的端口（如3121）对外开放，或 Nginx 反代正确配置。

龙虾回复说"没有权限"？

原因：你想用的功能（日历、文档等）对应的权限没有申请或没审批。

解决：回飞书开放平台「权限管理」，添加需要的权限，然后让管理员审批。

图片发不出来？

原因：缺少 im:resource 权限，或者图片文件太大（飞书限制约20MB）。

解决：确认权限已开启。大图片龙虾会自动压缩后再发。

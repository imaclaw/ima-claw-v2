1

创建 Skill 项目

按照标准目录结构创建你的 Skill。每个 Skill 必须包含一个 SKILL.md 文件作为入口。

2

编写 SKILL.md

描述技能的触发条件、执行流程、输入输出。这是龙虾理解和执行你技能的关键文档。

3

本地测试

在你的龙虾上安装测试：clawhub install ./your-skill，确保功能正常、无安全风险。

4

发布到 ClawHub

使用 clawhub publish 提交到社区市场。通过安全审核后自动上架。

📁 标准目录结构

my-awesome-skill/
├── SKILL.md ← 必需：技能描述和执行指令
├── README.md ← 推荐：使用说明和示例
├── scripts/ ← 可选：辅助脚本
│   └── run.sh
├── templates/ ← 可选：模板文件
└── assets/ ← 可选：图片等资源

审核标准

🔒

安全优先

不得包含恶意代码、数据窃取或危险系统命令。所有外部 API 调用需明确声明。

📝

文档完整

SKILL.md 必须清晰描述功能、触发条件和使用方法。无文档不审核。

⚡

功能明确

一个 Skill 做好一件事。避免功能过于庞杂或与已有 Skill 高度重复。

🧪

经过测试

提交前请充分本地测试，确保在不同场景下稳定运行。
